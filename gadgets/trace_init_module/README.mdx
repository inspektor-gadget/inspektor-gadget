# trace_init_module

The trace_init_module gadget emits events when processes invoke the `init_module()` or `finit_module()` syscalls to load kernel modules.

**Syscalls traced:**
- `init_module(module_image, len, param_values)` - loads a module from a memory buffer
- `finit_module(fd, param_values, flags)` - loads a module from a file descriptor

This gadget captures:
- For `init_module`: module size (`len`) and parameters (`param_values`)
- For `finit_module`: file descriptor (`fd`), resolved filepath, flags, and parameters (`param_values`)

**Note:** The `module_image` parameter from `init_module` is intentionally not captured.

## Requirements

- Minimum Kernel Version : 5.4

Running the gadget:

## Flags

### `--pid`

Show only events generated by processes with this pid

Default value: ""

### `--uid`

Show only events generated by processes with this uid

Default value: ""

## Guide

This guide shows how to use the `trace_init_module` gadget to detect when a
kernel module is loaded into the running kernel through the `init_module()` or
`finit_module()` syscalls.

Kernel modules are loaded from the host, so let's start the gadget first and
leave it running. With `ig`, pass `--host` so that events not belonging to a
container are also shown:

<Tabs groupId="env">
  <TabItem value="kubectl-gadget" label="kubectl gadget">
```bash
$ kubectl gadget run trace_init_module:%IG_TAG%
K8S.NODE        K8S.NAMESPACE K8S.PODNAME K8S.CONTAINERNAME COMM     PID     TID     TID     PARAM_VALUES LEN FD FLAGS FILEPATH                        SYSCALL
```
  </TabItem>

  <TabItem value="ig" label="ig">
```bash
$ sudo ig run trace_init_module:%IG_TAG% --host
RUNTIME.CONTAINERNAME COMM     PID     TID     TID     PARAM_VALUES LEN FD FLAGS FILEPATH                        SYSCALL
```
  </TabItem>
</Tabs>

Then, in another terminal, load a kernel module to generate an event. In this
example we load the `dummy` network module with a parameter:

<Tabs groupId="env">
  <TabItem value="kubectl-gadget" label="kubectl gadget">
Modules are loaded on the node, so run a privileged pod that mounts the node's
modules and loads it. Save the following as `modloader.yaml`:

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: modloader
spec:
  restartPolicy: Never
  containers:
  - name: modloader
    image: ubuntu
    securityContext:
      privileged: true
    command: ["bash", "-c", "apt-get update && apt-get install -y kmod && modprobe dummy numdummies=2"]
    volumeMounts:
    - name: modules
      mountPath: /lib/modules
      readOnly: true
  volumes:
  - name: modules
    hostPath:
      path: /lib/modules
```

```bash
$ kubectl apply -f modloader.yaml
pod/modloader created
```
  </TabItem>

  <TabItem value="ig" label="ig">
```bash
$ sudo modprobe dummy numdummies=2
```
  </TabItem>
</Tabs>

Going back to the gadget terminal, we can see the load event. It shows the
process that loaded the module (`modprobe`), the file descriptor and flags
passed to `finit_module()`, the resolved path of the module file, and the
parameters (`numdummies=2`):

<Tabs groupId="env">
  <TabItem value="kubectl-gadget" label="kubectl gadget">
```bash
K8S.NODE        K8S.NAMESPACE K8S.PODNAME K8S.CONTAINERNAME COMM     PID     TID     TID     PARAM_VALUES LEN FD FLAGS FILEPATH                        SYSCALL
minikube-docker default       modloader   modloader         modprobe 2323131 2323131 2320727 numdummies=2   0  3     4 /usr/lib/modules/6.17.0-35-gen… finit_module
```
  </TabItem>

  <TabItem value="ig" label="ig">
```bash
RUNTIME.CONTAINERNAME COMM     PID     TID     TID     PARAM_VALUES LEN FD FLAGS FILEPATH                        SYSCALL
                      modprobe 1992013 1992013 1992012 numdummies=0   0  3     4 /usr/lib/modules/6.17.0-35-gen… finit_module
```
  </TabItem>
</Tabs>

Finally, unload the module and clean up:

<Tabs groupId="env">
  <TabItem value="kubectl-gadget" label="kubectl gadget">
```bash
$ kubectl delete pod modloader
pod "modloader" deleted
```

To unload the module from the node, re-apply `modloader.yaml` with the command
changed to `modprobe -r dummy`, then delete that pod as well.
  </TabItem>

  <TabItem value="ig" label="ig">
```bash
$ sudo modprobe -r dummy
```
  </TabItem>
</Tabs>
