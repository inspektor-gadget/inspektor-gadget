The `snapshot_file` gadget shows open files for running processes.

## Requirements

- Minimum Kernel Version : *5.8

The gadget uses the `task_file` BPF iterator, available since Linux 5.8.

Classifying anonymous-inode file types (`BPF_MAP`, `BPF_LINK`, `BPF_PROG`,
`EVENTFD`, `TIMERFD`, `SIGNALFD`, `EPOLL`, `PERF_EVENT`, `USERFAULTFD`, `PIDFD`,
`IO_URING`, `FANOTIFY`, `INOTIFY`) additionally relies on `__ksym` weak data
relocations, which require Linux 5.11 or later and `CAP_SYSLOG` (to read symbol
addresses from `/proc/kallsyms`). When a required symbol is unavailable, files of
that type fall back to `OTHER` instead of their specific type; all other
functionality is unaffected.

*This is the minimal kernel version we have tried for this Gadget, however it's possible that it works with earlier versions.

## Flags

### `--type-mask`

Filter which file types to display using a bitmask. Each file type has a
bit; set a bit to include that type. By default all types are shown
(4294967295). Sum the values of the types you want to keep.

| Bit | Value | Type | Description |
|-----|-------|------|-------------|
| 0 | 1 | REGULAR | Regular file |
| 1 | 2 | SOCKET | Socket |
| 2 | 4 | PIPE | Pipe / FIFO |
| 3 | 8 | BPF_MAP | BPF map |
| 4 | 16 | DIRECTORY | Directory |
| 5 | 32 | CHAR_DEV | Character device |
| 6 | 64 | BLOCK_DEV | Block device |
| 7 | 128 | SYMLINK | Symbolic link |
| 8 | 256 | BPF_LINK | BPF link |
| 9 | 512 | BPF_PROG | BPF program |
| 10 | 1024 | EVENTFD | eventfd |
| 11 | 2048 | TIMERFD | timerfd |
| 12 | 4096 | SIGNALFD | signalfd |
| 13 | 8192 | EPOLL | epoll instance |
| 14 | 16384 | PERF_EVENT | perf event |
| 15 | 32768 | USERFAULTFD | userfaultfd |
| 16 | 65536 | PIDFD | pidfd |
| 17 | 131072 | IO_URING | io_uring |
| 18 | 262144 | FANOTIFY | fanotify |
| 19 | 524288 | INOTIFY | inotify |
| 20 | 1048576 | OTHER | Any other file type |

Examples:
- Regular files and directories only: 1 + 16 = 17
- BPF maps, links and programs: 8 + 256 + 512 = 776
- All types (default): 4294967295

Default value: "4294967295"

### `--paths`

Enable to get the complete file paths in the output. If disabled, only the file names will be shown.

Default value: "false"

## Guide

Run a pod / container:

<Tabs groupId="env">
<TabItem value="kubectl-gadget" label="kubectl gadget">
```bash
$ kubectl run --restart=Never --image=busybox test-snapshot-file -- sh -c 'echo hello > /tmp/demo.txt && tail -f /tmp/demo.txt'
pod/test-snapshot-file created
```

</TabItem>

<TabItem value="ig" label="ig">
```bash
$ docker run --name test-snapshot-file -d busybox /bin/sh -c 'echo hello > /tmp/demo.txt && tail -f /tmp/demo.txt'
...
```
</TabItem>
</Tabs>

Now, run the gadget and see how it shows the open file:

<Tabs groupId="env">
<TabItem value="kubectl-gadget" label="kubectl gadget">
```bash
$ kubectl gadget run snapshot_file:%IG_TAG%
K8S.NODE         K8S.NAMESPACE   K8S.PODNAME         K8S.CONTAINERNAME   COMM   PID   TID   FD   PATH      TYPE       FLAGS    
minikube-docker  default         test-snapshot-file  test-snapshot-file  tail   5432  5432  3    demo.txt  REGULAR    O_RDONLY  
^C
```
</TabItem>

<TabItem value="ig" label="ig">
```bash
$ sudo ig run snapshot_file:%IG_TAG% -c test-snapshot-file
RUNTIME.CONTAINERNAME             COMM               PID        TID   FD         PATH            TYPE            FLAGS              
test-snapshot-file                tail               5321       5321  3          demo.txt        REGULAR         O_RDONLY
^C
```
</TabItem>
</Tabs>

Finally, clean the system:

<Tabs groupId="env">
<TabItem value="kubectl-gadget" label="kubectl gadget">
```bash
$ kubectl delete pod test-snapshot-file
```
</TabItem>

<TabItem value="ig" label="ig">
```bash
$ docker rm -f test-snapshot-file
```
</TabItem>
</Tabs>
